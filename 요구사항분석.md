# 요구사항 분석

---

## Entity

### `Pokemon` (포켓몬)

- ID: 고유한 양의 정수
- 이름: 고유한 이름
- 타입: 1개 이상의 타입
- 키: 데시미터 단위의 측정값
- 몸무게: 헥토그램 단위의 측정값

### `PokemonType` (포켓몬 타입)

노말, 격투, 비행, 독, 땅, 바위, 벌레, 고스트, 강철, 
불꽃, 물, 풀, 전기, 에스퍼, 얼음, 드래곤, 악, 페어리, 
스텔라, 미지정, 섀도우

## Usecase

1. **포켓몬 목록 조회**

- 유저 조회 요청 시, 오름차순으로 일정한 수의 다음 포켓몬 목록을 조회한다.
- 예외 케이스
    - 오프라인: 네트워크 연결 확인 안내 + 재시도
    - 서버 오류: 사용자 재시도 안내
    

2. **포켓몬 상세 정보**

- 유저가 유효한 ID의 포켓몬을 선택 시, 해당 ID에 해당하는 상세 정보를 조회한다.
- 예외 케이스
    - 오프라인: 네트워크 연결 확인 안내 + 재시도
    - 서버 오류: 사용자 재시도 안내

3. **포켓몬 이미지 조회**
- 유효한 ID의 포켓몬을 통해 해당 포켓몬의 이미지를 조회한다.
- 예외케이스
    - 오프라인: 네트워크 연결 확인 안내 + 재시도
    - 서버 오류: 사용자 재시도 안내

## UI

**메인화면**

1. 포켓몬 이미지 그리드 뷰(스크롤 가능)를 통해 뷰에 출력 (줄 당 3개), 세로스크롤
2. 스크롤을 통해 다음 이미지 추가 출력
3. 이미지 클릭 시, 상세화면으로 이동
4. 이미지 로딩 상태 시, 인디케이터 표시
5. 이미지 로딩 실패 시, 재시도 새로고침 버튼을 통해 재시도 지원
6. 스크롤 로딩 시, 스크롤 뷰 하단에 인디케이터 표시
7. 에러 출력
    1. 현재 로딩된 데이터가 존재하지 않는 경우.
        1. 사용자 조작가능,  스크롤 뷰를 재시도 안내 뷰(”해결방법 메세지”)로 교체
        2. 사용자의 조작 불가능, 스크롤 뷰를 재시도 안내 뷰(“알 수 없는 에러”)로 교체
    2. 스크롤을 통한 추가 로딩의 경우.
        1. 사용자 조작가능,  알럿을 통해 안내 메세지와 재시도 알럿 출력
        2. 사용자의 조작 불가능, “알 수 없는 에러” 메세지와 재시도 알럿 출력

**상세화면** 

1. (포켓몬 이미지, 포켓몬 번호, 이름(한글), 타입, 키, 몸무게 표시)
2. 이미지 로딩 실패 시, 재시도 버튼
3. 에러 출력
    1. 사용자 조작가능, 알럿을 통해 안내 메세지와 재시도 알럿 출력
    2. 사용자의 조작 불가능, “알 수 없는 에러” 메세지와 재시도 알럿 출력

## API

- **NamedAPIResourceList(Group)**
    
    `https://pokeapi.co/api/v2/{endpoint}//?limit={limit}&offset={offset}`
    
    | Name | Description | Type |
    | --- | --- | --- |
    | **count** | The total number of resources available from this API. | *integer* |
    | **next** | The URL for the next page in the list. | *string* |
    | **previous** | The URL for the previous page in the list. | *string* |
    | **results** | A list of named API resources. | *[NamedAPIResource]* |
- **NamedAPIResource**
    
    
    | Name | Description | Type |
    | --- | --- | --- |
    | **name** | The name of the referenced resource. | *string* |
    | **url** | The URL of the referenced resource. | *string* |
